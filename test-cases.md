# Design Pattern Miner 测试脚本

> 用于测试技能的核心功能

---

## 测试用例 1: GitHub 项目分析

**输入**：
```
分析这个项目的设计思想：https://github.com/AlexAnys/awesome-openclaw-usecases-zh
```

**期望输出**：
- ✅ 询问用户配置（输出格式、深度、内容）
- ✅ 项目概述完整
- ✅ 核心设计思想 3-5 个
- ✅ 架构拆解清晰
- ✅ 可复用方法论 5-10 个
- ✅ 优缺点分析
- ✅ 适用场景明确

**验证标准**：
- 用户能在 10 分钟内理解项目核心
- 提取的可复用点可直接用于用户项目
- 不涉及源码复制

## 测试用例 3: 默认配置测试

**输入**：
```
分析这个项目，默认配置：https://github.com/xxx/xxx
```

**期望输出**：
- ✅ 跳过配置询问
- ✅ 使用默认配置（全部格式 + 概要级 + 架构 + 产品 + 可复用）
- ✅ 输出完整报告

**验证标准**：
- 正确识别"默认配置"关键词
- 使用正确的默认配置

## 测试用例 4: 自定义配置测试

**输入**：
```
分析这个项目，输出格式全部，深度详细级，内容架构设计 + 产品思维
https://github.com/xxx/xxx
```

**期望输出**：
- ✅ 确认用户配置
- ✅ 按配置输出（全部格式 + 详细级 + 架构 + 产品）
- ✅ 不包含其他内容（如核心算法、实现技巧）

**验证标准**：
- 正确解析配置选项
- 严格按配置输出

## 测试流程

### 1. 准备阶段

- [ ] 确认技能已安装
- [ ] 确认 CoPaw 已重启
- [ ] 准备测试 URL

### 2. 执行测试

- [ ] 运行测试用例 1（GitHub 项目）
- [ ] 运行测试用例 2（微信文章）
- [ ] 运行测试用例 3（默认配置）
- [ ] 运行测试用例 4（自定义配置）
- [ ] 运行测试用例 5（简化配置）
- [ ] 运行测试用例 6（对比分析）
- [ ] 运行测试用例 7（小红书）
- [ ] 运行测试用例 8（授权边界）

### 3. 验证结果

- [ ] 检查输出格式
- [ ] 检查配置解析
- [ ] 检查提取质量
- [ ] 检查授权边界

### 4. 记录问题

- [ ] 记录所有问题
- [ ] 分类问题（严重/一般/建议）
- [ ] 提出改进建议
