# lark-design-pattern-miner v1.0.0 - 设计模式挖掘机

> **从任何开源项目/文章中提取设计思想、实现逻辑和核心架构 —— 提取思想，不复用代码；把选择交给用户。**

## 🎯 定位

**Design Pattern Miner 是一个**开发学习工具，帮你快速从任何开源项目/技术文章中挖出可复用的设计思想：

- 给你一个 GitHub 链接 / 文章链接 / 笔记链接
- 它帮你提取：核心架构设计、实现技巧、产品思维、可复用方法论
- **核心理念**：提取思想，不复用代码；把选择交给用户
- **支持多输入源**：GitHub 项目、微信文章、小红书笔记、技术博客
- **完全可配置**：输出格式、提取深度、输出内容全部由你决定

**适合场景**：
- ✅ 学习优秀开源项目 → 快速提取设计思想，不用一行行读代码
- ✅ 技术文章精读 → 结构化提取核心要点
- ✅ 借鉴设计模式 → 挖出可复用的架构思路，用到自己项目中
- ✅ 技术预研 → 快速了解多个项目的设计对比
- ✅ 学习新思路 → 从优秀项目中学习，站在巨人肩膀上

## 🧠 策略引擎

### 核心工作流

```
  用户提供 URL（GitHub/文章/笔记/博客）
    ↓
  Design Pattern Miner 识别来源类型
    ↓
  抓取内容（README / 源码 / 文章正文）
    ↓
  询问用户配置（输出格式 / 提取深度 / 输出内容）
    ↓
  AI 深度分析提取：
  - 拆解核心架构
  - 提取设计思想
  - 找出可复用点
  - 生成结构化报告
    ↓
  按用户配置输出：
  - Markdown 文档
  - Mermaid 流程图 / 架构图
  - CoPaw / 飞书 CLI 技能模板
  - 完整分析报告
    ↓
  ✅ 完成！你得到一份结构化的设计思想提取报告
```

### 可配置选项

#### 📋 输出格式（可多选）

| 选项 | 格式 | 适用场景 |
|------|------|----------|
| **A** | 结构化文档（Markdown） | 快速阅读、保存文档 |
| **B** | 流程图 + 文字说明（Mermaid） | 理解数据流、架构可视化 |
| **C** | 技能模板（CoPaw / 飞书 CLI 格式） | 提取后直接创建技能 |
| **D** | 架构 diagrams | 理解系统架构 |
| **E** | 全部格式 | 完整分析 |

#### 📊 提取深度（可多选）

| 选项 | 深度 | 理解时间 | 适合场景 |
|------|------|----------|----------|
| **A** | 概要级 | 10 分钟 | 快速了解、多项目对比 |
| **B** | 详细级 | 1 小时 | 深入学习、技术决策 |
| **C** | 可执行级 | 可复现 | 准备复现、技术预研 |

#### 📝 输出内容（可多选）

| 选项 | 内容 | 提取重点 |
|------|------|----------|
| **A** | 架构设计 | 模块划分、数据流、接口设计 |
| **B** | 核心算法 | 关键算法逻辑、复杂度、优化 |
| **C** | 实现技巧 | 代码技巧、性能优化、边界处理 |
| **D** | 产品思维 | 设计原因、痛点解决、目标用户 |
| **E** | 可复用组件 | 可借鉴部分、需改造部分 |

## 优缺点分析

- ...
### 缺点
- ...

## 📋 配置命令

### 完整语法

```
分析 [URL]，输出格式 [选项]，深度 [选项]，内容 [选项]
```

### 配置速查表

| 配置项 | 选项 | 默认值 | 示例 |
|-------|------|--------|------|
| **输出格式** | A/B/C/D/E（可多选） | E（全部） | `输出格式 A+B` |
| **提取深度** | A/B/C（可多选） | A（概要级） | `深度 详细级` |
| **输出内容** | A/B/C/D/E（可多选） | A+D+E | `内容 架构设计 + 产品思维` |

### 快捷命令

| 命令 | 含义 |
|------|------|
| `默认配置` | 使用默认配置 |
| `全部` | 所有选项全选 |

## 🔒 安全边界

### 授权边界

严格遵守：
- ✅ **提取设计思想**（允许）
- ✅ **学习借鉴**（允许）
- ❌ **直接复制源码**（需要遵守原项目协议）
- ❌ **声称原创**（不道德）

**使用原则**：提取思想，不复用代码；注明灵感来源；遵守原项目协议。

### 提取限制

- 部分项目文档不全 → 影响分析质量
- 私有仓库无法访问 → 需要你提供访问权限
- 需要登录的网站 → 需要 Cookie 才能抓取

## 💡 设计哲学

- **最小完备** - 只做提取和分析，不改变原项目
- **用户做主** - 所有配置由用户选择，把选择权交给用户
- **合规优先** - 只提取思想，不碰代码，遵守版权
- **渐进披露** - 从概要到详细，用户按需索取
